## 🚀 Jarvis – Next Phase (Umsetzung v2)

## Status

- Erledigt im Code:
  - UI verbessert: Chat-Bubbles, Thinking-Delay, Theme-Schalter, zusätzliche Animationen
  - Responses verbessert: mehrere Varianten pro Command, zufällige Auswahl
  - Response-System erweitert: `text`, `link`, `list`, `error`
  - Context-System eingebaut: `lastResult` wird gespeichert und für Folge-Rechnungen genutzt
  - Error Handling beruhigt: keine technischen Fehler mehr in der UI
  - Quick Actions nach jeder Antwort abgesichert
  - Logging lokal eingebaut über `data/interaction-log.json`
- Noch nicht umgesetzt:
  - Ollama-Installation
  - AI-Routing über lokales Modell

---

### 1. 🎨 UI verbessern (Feeling first)

* Chat Bubbles sauber stylen (User rechts, Jarvis links)
* Ladeanimation einbauen:

  * „Jarvis denkt…“ (300–800ms Delay)
* Dark Mode + einfache Animationen hinzufügen

---

### 2. 🧠 Responses verbessern

* Keine statischen Antworten mehr
* Pro Command mehrere Antworten definieren
* Zufällig auswählen (Variation)

---

### 3. 📦 Response System erweitern

Backend:

* Einheitliches Format nutzen:

```json
{
  "type": "text",
  "content": "..."
}
```

Erweitern um:

* `link`
* `list`
* `error`

Frontend:

* Je nach `type` unterschiedlich rendern

---

### 4. 🧠 Context System einbauen

* Variable speichern:

  * `lastResult`
* Beispiel:

  * „rechne 5+5“ → speichere 10
  * „mal 2“ → nutze 10 weiter

---

### 5. ⚠️ Error Handling verbessern

* Keine technischen Fehler anzeigen
* Stattdessen:

  * „Das habe ich nicht verstanden“
  * „Versuch es nochmal anders“

---

### 6. ⚡ Quick Actions im UI

Nach jeder Antwort anzeigen:

* Buttons für:

  * Zeit anzeigen
  * Rechnen
  * Amazon Suche

---

### 7. 📝 Logging System

* Speichere lokal:

  * letzte Inputs
  * verwendete Commands
* Ziel:

  * später Optimierung / AI Training

---

### 8. 🤖 Ollama vorbereiten (noch NICHT voll nutzen)

Status: manuell offen

* Ollama installieren
* lokal testen:

```bash
ollama run llama3
```

---

### 9. 🧠 AI nur für Routing nutzen

Status: bewusst noch offen

* AI entscheidet:

  * welcher Command genutzt wird
* Commands bleiben weiterhin manuell

---

### 10. 🔥 Clean Code halten

* Trennung beibehalten:

  * Commands
  * Parser
  * Response System
* Keine Logik mischen

---

## 🎯 Ziel dieser Phase

* System fühlt sich wie echtes Produkt an
* UI + UX deutlich besser
* Vorbereitung für AI ohne Chaos

## Manuell zu tun

1. Jarvis lokal starten und UI kurz durchklicken:
  - Zeit
  - Rechnen
  - Folge-Rechnung wie `mal 2`
  - Amazon-Suche
  - URL öffnen
  - Notizenliste
2. Falls du Ollama wirklich vorbereiten willst, Ollama separat installieren und in einer Shell testen:

```bash
ollama run llama3
```

3. Wenn du AI-Routing als nächsten Schritt willst, entscheide dich für ein Modell und ob Routing lokal nur als Fallback oder als Standard laufen soll.
